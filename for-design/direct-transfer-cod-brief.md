# สเปค: รับชำระนอกแพลตฟอร์ม — ① ลูกค้าโอนตรงเข้าบริษัท ② COD ผ่านขนส่ง

> โน้ตจากเจ้าของ 2026-07-23 · **ยังไม่ทำ** — รอไฟล์ตัวอย่าง (BigSeller export + รายงานขนส่ง COD)
> ที่อยู่ปลายทาง: หน้า **Bank Reconciliation** → แท็บใหม่ **ข้าง "🛒 ถอน Marketplace"**

---

## ภาพรวม — การรับเงินของบริษัทมี 3 แบบ

| แบบ | ช่องทางขาย | เงินเข้าอย่างไร | ค่าธรรมเนียม | เช็ค/BQ | สถานะระบบ |
|---|---|---|---|---|---|
| ผ่านแพลตฟอร์ม | Shopee / TikTok / Lazada | กระเป๋าแพลตฟอร์ม → ถอนเข้าแบงค์เป็นก้อน | มี (หักในกระเป๋า) | มี (เช็ครับ + BQ) | ✅ ทำแล้ว (แท็บถอน Marketplace) |
| **① โอนตรง** | FACE / LINE / Dealer | **ลูกค้าโอนเข้าบัญชีบริษัทโดยตรง ทีละรายการ** | **ไม่มี** | **ไม่มี** | ⬜ ต้องทำ |
| **② COD** | FACE / LINE (ไม่ผ่านแพลตฟอร์ม) | ขนส่งเก็บเงินปลายทาง → โอนคืนเป็นก้อน **หลังหักค่าบริการ** | มี (ค่าบริการขนส่ง) | มี (เช็ค COD + BQ) | ⬜ ต้องทำ |

## ① แท็บใหม่: "ลูกค้าโอนตรงเข้าบริษัท"

### ขอบเขต
ออเดอร์จาก BigSeller ที่ **ไม่ใช่แพลตฟอร์ม** (FACE/LINE/Dealer) **และ** ประเภทการชำระเงิน = **โอนตรงเข้าบัญชี** (ไม่ใช่ COD)
→ ระบบคัดออเดอร์พวกนี้มารอในแท็บนี้ เพื่อจับคู่กับ **เงินเข้าจริงในสเตทเมนต์** (`brec_bank_rows` ฝั่ง deposit)

### ตัวอย่างเคส
คีย์ขาย 23/07/2026 ยอด 390 · ประเภท "โอนเงินตรงเข้าบัญชี" · ฝ่ายขายระบุว่าโอนวันที่ 23/07/2026
→ ต้องบอกได้ว่า **ออเดอร์นี้เงินเข้าแล้วหรือยัง** และเข้าด้วยรายการไหนในสเตทเมนต์

### ★ ข้อควรระวังที่สุด (เจ้าของย้ำ) — ห้ามจับคู่มั่ว
- วันเดียวกันอาจมีเงินเข้า **ยอด 390 หลายครั้ง** → ยอด+วันอย่างเดียวไม่พอ
- เคสไม่มีปัญหา: คีย์ 5 ออเดอร์ ยอด 390 · เงินเข้า 390 จำนวน 5 ครั้ง → นับตรงกัน
- **เคสอันตราย: คีย์ 5 แต่เงินเข้า 4** → ถ้าจับมั่ว = **ออเดอร์ที่ยังไม่จ่ายจะถูกนับว่าจ่ายแล้ว → ลูกหนี้คงค้างผิด**
- ⇒ ถ้าไม่ชัดเจน **ห้าม auto-confirm** ต้องปล่อยให้ค้างเป็น "ต้องตรวจ" ให้คนเลือกคู่เอง (ปกติต้องเปิดสลิปในระบบดู เพราะ Excel ที่ export ออกมาไม่ละเอียดพอ)

### กติกาจับคู่ที่เสนอ (ยืนยันกับเจ้าของก่อนทำ)
1. **ชัดเจน (เสนอให้ยืนยันได้):** วันตรง + ยอดตรง + **ชื่อผู้โอนในสเตทเมนต์ใกล้เคียงชื่อลูกค้า** และมีคู่ที่เป็นไปได้ **เพียงคู่เดียว**
2. **กำกวม (ต้องเลือกเอง):** วัน+ยอดตรง แต่มีหลายคู่ที่เป็นไปได้ (เช่น 390 เข้า 4 ครั้ง กับ 5 ออเดอร์) → โชว์ทั้งกลุ่ม ให้คนจับ 1:1 เอง + เตือนว่า "เงินเข้าน้อยกว่าออเดอร์ N รายการ"
3. **ไม่พบ:** ยังไม่มีเงินเข้าที่เข้าเกณฑ์ → คงสถานะ "รอเงินเข้า"
4. เผื่อวัน ±1–2 วัน ได้ (ลูกค้าโอนคนละวันกับที่ฝ่ายขายกรอก) — แต่ต้องขึ้นธงว่า "คนละวัน"

### หลังจับคู่ → ปุ่มติ๊ก "ตรวจ/ยืนยัน" (การเงิน)
- ติ๊กยืนยัน = บันทึกว่า **เงินเข้าบริษัทแล้วจริง** (ลง `bank_in_date` / `bank_amount` / `bank_matched` + ผู้ยืนยัน + เวลา)
- ยืนยันแล้ว → **ส่งต่อไปหน้า "ระบบงานขาย · 2. รับชำระเงิน"** ให้ส่งออกรับชำระ (RE) ได้
- **ความต่างของ RE แบบโอนตรง เทียบกับผ่านแพลตฟอร์ม:**
  - รับเงินเข้าบัญชีตรง → **ไม่ต้องมีเช็ครับ / รหัสเช็ค / BQ**
  - **ไม่มีค่าธรรมเนียม** → เงินเข้าสุทธิ = ยอด IV เต็ม (ไม่มีรายการ "ค่าธรรมเนียมจ่ายล่วงหน้า")

### ข้อมูลที่ยังขาด (ต้องเก็บเพิ่มตอนนำเข้า BigSeller)
ตอนนี้ `ORD_CH.bigseller` (index.html) เก็บแค่ ออเดอร์/สถานะ/แพลตฟอร์ม/ร้าน/วันที่/สินค้า/SKU/จำนวน/ราคา/ค่าส่ง/ส่วนลด
**ยังไม่เก็บ:**
- ประเภทการชำระเงิน (โอนตรง / COD)  ← ตัวคัดกรองหลักของแท็บนี้
- ชื่อผู้ซื้อ / ชื่อผู้โอน  ← ใช้เทียบกับชื่อในสเตทเมนต์ (`order_ledger.customer` ตอนนี้ว่างสำหรับออเดอร์ที่มาจาก BigSeller — เติมเฉพาะตอน ingest จาก 723-5)
- วันที่โอนที่ฝ่ายขายกรอก + หมายเหตุ/สลิป

⇒ ต้องทำ: เพิ่ม column map ใน `ORD_CH.bigseller` + migration เพิ่มคอลัมน์ใน `order_ledger`
(เช่น `pay_method`, `buyer_name`, `pay_transfer_date`, `pay_note`)
**⚠ ต้องขอไฟล์ BigSeller ตัวอย่างจากเจ้าของก่อน** เพื่อดูชื่อคอลัมน์จริง

## ② COD ผ่านขนส่ง (ขาย FACE/LINE แบบเก็บเงินปลายทาง)

### กระบวนการจริง
1. ลูกค้าจ่ายเงินให้ขนส่งตอนรับของ
2. ขนส่งรวบรวม → **โอนคืนบริษัทเป็นก้อน หลังหักค่าบริการ**
3. บัญชีรับชำระ **แบบเดียวกับแพลตฟอร์มออนไลน์** = รับชำระผ่าน **รหัสเช็คที่เป็น COD**
4. ดึงรายงานจากขนส่งมาแมพว่า **เงินก้อนที่โอนมา ประกอบด้วยออเดอร์ไหน / RE ไหนบ้าง**
5. ผ่านเช็คเข้าบัญชีด้วย**ยอดรวม** — ค่าใช้จ่าย/ค่าบริการหักที่หน้าผ่านเช็ค (**BQ**)

### หมายเหตุงาน
- โครงเหมือน "ถอน Marketplace" เป๊ะ (เงินเข้าเป็นก้อน = หลายออเดอร์ − ค่าบริการ) → น่าจะทำเป็นอีก channel/แท็บที่ reuse `bmpGroupWithdrawals` + gen BQ + จับเลขที่เช็ค ได้เลย
- **รอไฟล์ตัวอย่าง "รายงานที่เอาไว้แมพออเดอร์ COD" จากเจ้าของ** (ยังไม่รู้ว่าเป็นของขนส่งเจ้าไหน / รูปแบบคอลัมน์)
