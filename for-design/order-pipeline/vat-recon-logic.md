# VAT / Recon Logic — ฐานภาษีขาย (สเปคฉบับเต็มสำหรับ recon engine)

> ที่มา: เจ้าของส่งไฟล์ให้ AI ตรวจจากข้อมูลจริง (01–18/06/2569) แล้วสรุปเป็นกฎ
> สถานะยืนยัน: Shopee 1,401/1,401 ✅ · Lazada 29/29 ✅ (sample เล็ก) · TikTok 128/128 (QI) ✅ · MBARK TikTok ยังไม่เช็ค
> ⚠️ ก่อนล็อกใช้จริง: แนะนำรัน MBARK TikTok + Shopee/Lazada อีกเดือนเต็มให้ผลต่าง = 0 ทุกตัว

## 1. หลักการฐานภาษีขาย (กฎแม่บท)
**ฐานภาษีขาย (รวม VAT) = ยอดสินค้าสุทธิเฉพาะส่วนที่ผู้ขายลด + ค่าส่งที่ผู้ซื้อจ่าย**
- **หัก:** ส่วนลดที่ "ผู้ขาย" ออกเอง
- **ไม่หัก:** ส่วนลดที่ "แพลตฟอร์ม" ออก (ผู้ขายยังได้เงินส่วนนี้เต็มจากแพลตฟอร์ม)
- **ไม่รวม:** ค่าธรรมเนียม, คอมมิชชัน, VAT ค่าธรรมเนียม, เงินโอนสุทธิ (settlement)

## 2. การคีย์ราย SKU (1 บรรทัด BigSeller = 1 บรรทัด Express)
แต่ละบรรทัด SKU: รหัสสินค้า + จำนวน + ราคา/หน่วยสุทธิ
ราคา/หน่วยสุทธิ — ใช้คนละช่องตามแพลตฟอร์ม (ห้ามสลับ):
```
IF Shopee:
    qty        = จำนวน
    unit_price = ราคา − (Voucher ของร้านค้า ÷ จำนวน)
ELIF Lazada:
    qty        = จำนวน                  # = 1 เสมอ
    unit_price = ราคา − (Voucher ของร้านค้า ÷ จำนวน)
ELIF TikTok:
    qty        = จำนวน
    unit_price = ราคาสินค้าเดิม − Voucher ของร้านค้า   # ทั้งคู่เป็นค่า "ต่อหน่วย" อยู่แล้ว
line_total = unit_price × qty
```
**เหตุผล TikTok ใช้คนละช่อง:** ช่อง "ราคา" ของ TikTok = ราคาหลังหักส่วนลดแพลตฟอร์มไปแล้ว (ต่ำกว่าจริง) · Shopee/Lazada ช่อง "ราคา" สะอาด → ใช้ตรงๆ

## 3. ค่าส่ง (คีย์แยก ไม่กระจายลง SKU)
ค่าส่ง = ระดับออเดอร์ → 1 บรรทัด/ออเดอร์ (รหัส "ค่าขนส่ง")
- Shopee: ค่าจัดส่ง (ซ้ำทุกบรรทัด → เอาค่าเดียว/ออเดอร์)
- Lazada: ค่าจัดส่ง (แยกราย SKU → รวม/ออเดอร์)
- TikTok: ค่าจัดส่ง (ระดับออเดอร์ → เอาค่าเดียว/ออเดอร์)

## 4. กรองก่อนคำนวณเสมอ (Pre-process)
1. ตัดออเดอร์ยกเลิก/คืน: status ∈ {ยกเลิกแล้ว, การคืนเงิน/คืนสินค้า, Canceled, ...} → ข้าม
2. group by หมายเลขคำสั่งซื้อ
3. แต่ละ SKU = 1 บรรทัด, qty = จำนวนของบรรทัดนั้น

## 5. ช่องต้องห้าม (อย่าให้เว็บแตะ)
| ช่อง BigSeller | เหตุผลห้ามใช้ |
|---|---|
| ยอดรวมคำสั่งซื้อ | ยอดหลัง settlement ไม่ใช่ฐานภาษี |
| Voucher | ส่วนลดแพลตฟอร์ม — ห้ามหักทุกกรณี |
| ส่วนลดผู้ขาย | ค่าขยะ/ไม่คงเส้นคงวา (Shopee บวมเกิน, TikTok=0) |
| ราคาสินค้าเดิม (Shopee/Lazada) | Shopee=ราคาป้าย, Lazada=0/เพี้ยน → ใช้ "ราคา" แทน |
| ราคา (TikTok) | หักส่วนลดแพลตฟอร์มไปแล้ว → ใช้ ราคาสินค้าเดิม − Voucherร้านค้า แทน |

## 6. ตรวจยอด 2 ฝ่าย (Validation) — |ผลต่าง| ≥ 0.01 → flag
**ฝ่ายแพลตฟอร์ม (source of truth):**
- Shopee: `ราคาขายสุทธิ − โค้ดส่วนลดชำระโดยผู้ขาย + ค่าจัดส่งที่ชำระโดยผู้ซื้อ`
- Lazada: `unitPrice + sellerDiscountTotal + shippingFee`   (sellerDiscount เป็นค่าลบ)
- TikTok: `(SKU Subtotal Before Discount − SKU Seller Discount) + Shipping Fee After Discount`

**ฝ่าย BigSeller:** สูตรข้อ 2 + ข้อ 3

### ตารางวินิจฉัยเมื่อไม่ตรง
| อาการ | สาเหตุ | วิธีแก้ |
|---|---|---|
| ผลต่าง = ส่วนลดแพลตฟอร์ม (TikTok) | เผลอใช้ "ราคา" | เปลี่ยนเป็น ราคาสินค้าเดิม − Voucherร้านค้า |
| ผลต่าง = (qty−1) × ราคา/หน่วย | ลืม × จำนวน (TikTok per-unit) | คูณจำนวน |
| BigSeller สูงเกินมาก | ใช้ "ราคาสินค้าเดิม" กับ Shopee/Lazada | เปลี่ยนเป็น "ราคา" |
| มียอดทั้งที่ควรเป็น 0 | ออเดอร์ยกเลิกไม่ได้ถูกกรอง | กรอง status ก่อน |

## กฎจำสั้น
Shopee/Lazada ใช้ **ราคา** · TikTok ใช้ **ราคาสินค้าเดิม − Voucherร้านค้า** · ทุกตัวหักเฉพาะ **Voucher ของร้านค้า** · **ห้ามแตะ Voucher (แพลตฟอร์ม)** · บวกค่าส่งแยกบรรทัด · กรองยกเลิกก่อน
